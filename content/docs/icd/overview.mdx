## How to use the API

Provide a single clinical note as input. Use **`extract-conditions`** to get a list of conditions mentioned in the note, or **`classify-note`** to get ICD‑10 codes that represent those conditions (both generic and complete/specific when available). This is typically used after a visit when converting documentation into billable codes.

### Input Parameters

| Field                                                  | Type     | Description |
|-------------------------------------------------------|----------|-------------|
| `clinical_note`<span style={{ color: "red" }}>*</span> | `string` | Full, plain‑text clinical note to analyze. |

### Output

- **extract-conditions**  
  Returns: `conditions: string[]` — medical conditions/diseases mentioned in the note.

- **classify-note**  
  Returns: `icd10: string[]` — ICD‑10 codes corresponding to the note’s conditions.  
  May include both **generic** (e.g., `A00`) and **complete/specific** codes (e.g., `A00.0`).  

## How it works

The ICD API routes your clinical note through Avey’s coding pipeline. Avey parses the note, identifies clinical conditions, and classifies them into standardized ICD‑10 codes using Avey’s internal ICD Engine.
