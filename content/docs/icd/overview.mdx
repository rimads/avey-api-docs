The ICD API transforms unstructured clinical text into structured, standardized data by identifying medical concepts and assigning them corresponding ICD-10-CM codes.  
It streamlines coding workflows, reduces manual effort, and ensures consistency across healthcare systems.

## How to use the API
Call the ICD API to either extract conditions or return ICD-10-CM codes.

<Cards>
  <Card title="Extract">
    Send free text and receive a list of identified clinical conditions.
  </Card>
  <Card title="Classify">
    Send free text and receive ICD-10-CM codes for the conditions in the text.
  </Card>
</Cards>

<Callout type="info">Classification may include both generic and complete/specific ICD‑10-CM codes when available.</Callout>

For complete specifications and live testing, visit the 
<a href="#api-reference"

## Architecture

The ICD API routes your clinical note through Avey’s coding pipeline. Avey parses the note, identifies clinical conditions, and classifies them into standardized ICD‑10 codes using Avey’s internal ICD Engine.
